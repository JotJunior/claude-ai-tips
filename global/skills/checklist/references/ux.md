# Dominio: UX

Items exemplo para checklist de qualidade de requisitos UX.

## Hierarquia Visual

- Sao os requisitos de hierarquia visual definidos com criterios mensuraveis? [Clareza, Spec §FR-1]
- E 'exibicao proeminente' quantificado com sizing/positioning especificos? [Clareza]

## Estados de Interacao

- Sao requisitos de estado de interacao (hover, focus, active) consistentemente definidos? [Consistencia]
- Estados de loading sao especificados para operacoes assincronas? [Completude]
- Estados vazios (empty states) tem requisitos definidos? [Cobertura, Gap]

## Acessibilidade

- Sao requisitos de acessibilidade especificados para todos os elementos interativos? [Cobertura, Gap]
- Navegacao por teclado e definida para fluxos criticos? [Cobertura]
- Contraste de cor atende a WCAG 2.1 AA? [Conformidade]
- Leitores de tela tem labels/aria adequados? [Cobertura]

## Edge Cases UX

- E o fallback definido quando imagens falham ao carregar? [Edge Case, Gap]
- Mensagens de erro seguem padrao consistente? [Consistencia]
- Comportamento em conexao lenta/offline e especificado? [Edge Case]

## Responsividade

- Breakpoints sao definidos com criterios mensuraveis? [Clareza]
- Layout em mobile tem requisitos distintos do desktop? [Cobertura]
- Touch targets atendem tamanho minimo (~44px)? [Clareza]

## Microinteracoes

- Animacoes sao especificadas com duracao e easing? [Clareza]
- Feedback tatil/visual em acoes destrutivas e definido? [Completude]
